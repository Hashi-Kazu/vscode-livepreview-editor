# Live Preview Editor 要求仕様書（原本）

> この文書は要求の**原本**（散文）。開発時の読み書きは USDM 版 [requirements-usdm.md](requirements-usdm.md) を正として行う。

## 目的

Obsidian の Live Preview に近い、Markdown ファイル用の 1 枚ビュー編集エディタを VS Code 上で提供する。記法を保持したまま見た目を装飾し、カーソルがある行・要素では生の Markdown 記法を表示する。

## 背景

VS Code 標準の Markdown は「エディタ＋プレビューの 2 ペイン」が基本で、Obsidian のような単一ビューでの編集体験が得られない。本拡張は CodeMirror 6 を Webview に埋め込み、装飾と生記法をカーソル位置に応じて切り替えることでこの体験を実現する。

## 要求の概要

1. **基本記法のライブプレビュー** — 太字・見出し・リスト等を装飾表示し、カーソル行は生記法を表示する。
2. **拡張記法** — 斜体・インラインコード・引用・リンク・画像・コードブロック・表に対応する。
3. **ソース／プレビュー切り替え** — 同一エディタ上で装飾の ON/OFF を切り替える。
4. **同期** — Webview と TextDocument を双方向同期し、外部変更を検知して再同期する。
5. **エッジケース・品質** — ネストリスト、コードブロック内の誤装飾防止、IME、大ファイル性能、Undo/Redo、エラーフォールバック。
6. **設定** — 装飾 ON/OFF・フォントサイズ。
7. **Wikilink** — `[[ノート名]]` / `[[ノート名|別名]]` を装飾表示し、クリックで対象ファイルへ遷移する。（Obsidian 互換のリンク機能）
8. **タスクチェックボックス** — `- [ ]` / `- [x]` をチェックボックスとして表示し、クリックで完了状態を切り替える。

## スコープ外（現時点）

- グラフビュー（ノート関連性の可視化）
- バックリンクパネル
- AI 補助（Smart Composer 相当の RAG／LLM チャット）
- プラグイン拡張機構
