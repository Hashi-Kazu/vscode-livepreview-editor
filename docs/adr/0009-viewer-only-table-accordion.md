# ADR-0009: テーブル・アコーディオンのビューア専用化

- **ステータス**: 採択済み（v1.12.0 で採択）
- **確信度**: 高（コミット 05bb581・architecture.md・CLAUDE.md から明示的に確認）

## コンテキスト

v1.12.0 以前、テーブルと `<details>` アコーディオンはカーソルが入ったときに「生の Markdown / HTML」として編集できるライブ編集モードが存在していた。しかしこの方式には複数の問題があった:

- ブロックウィジェットの高さ会計（CodeMirror の `estimatedHeight`）が実際の描画高さとずれ、クリック位置がずれるバグが多発
- テーブルセルのインライン記法（太字・斜体等）のカーソル行検出が複雑になる
- アコーディオン（HTML タグ）の「生表示 ↔ ウィジェット切り替え」は実装の難易度が高く不具合が継続

## 決定

テーブル（`table-block` ウィジェット）と `<details>` アコーディオン（`details-block` ウィジェット）を **ビューア専用** とする:

- カーソルが中にあっても常にウィジェット描画を行う（生テキストに戻さない）
- インライン編集は廃止。編集したい場合は標準ソースエディタで行う

ただしテーブルはカーソルが入ったときのみ生テキスト表示へ戻す挙動を v1.12.0 でも残している（`blockHasCursor` による分岐）。アコーディオンのみ完全ビューア専用（`R-27-03`）。

**[要確認]**: テーブルについてもビューア専用にする計画があったか、またはカーソル時の生表示を意図的に残したか？

## 理由

- ブロックウィジェットの高さ会計ずれは CodeMirror の仕様上の制約であり、`requestMeasure` を使っても完全解消が困難（v1.15.x の修正履歴が示す）
- ビューア専用にすることで「ウィジェット ↔ 生テキスト切り替え」時の状態管理が不要になる
- アコーディオンは HTML タグを直接編集する必然性が薄い。標準エディタで編集すれば足りる

## 捨てた選択肢

- **カーソル行で生表示に戻す（ライブ編集）**: クリック位置ずれ・高さ計算ずれのバグが繰り返し発生（v1.13.x〜v1.16.x の修正コミット多数）
- **Obsidian 方式の完全ライブ編集**: 高さ会計の完全実装が困難でリグレッションが多い
